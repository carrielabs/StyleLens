# Design Standard

## 模板定位

面向管理层汇报、商业复盘、市场分析和战略推演的咨询级滚动数据报告模板。视觉方向强调结论先行、白底克制、图表与洞察左右分栏、细线分隔和高可读性。

## 核心原则

- 结论先行：模块标题必须是一句完整结论。
- 极度克制：避免大面积彩色背景、弥散阴影、大圆角和毛玻璃。
- 信息分栏：优先采用左侧图表 / 数据、右侧核心洞察的结构。
- 引导线：使用细实线分隔模块，使用细线或虚线组织图表与说明关系。

## 色彩规范

- 主背景：`#FFFFFF`
- 页面弱背景：`#FAFAFA`
- 弱强调背景：`#F8F9FA`、`#F0F4F8`
- 主文本：`#111827`
- 正文文本：`#4B5563`
- 次级文本：`#6B7280`
- 主强调色：`#0F2C59`
- 次强调色：`#00B4D8`
- 辅助灰：`#D1D5DB`
- 风险色：`#991B1B`
- 边框：`#E5E7EB`

## 字体规范

- 正文字体：`PingFang SC`、`Microsoft YaHei`、Helvetica、Arial、sans-serif。
- 核心结论字体：`Noto Serif SC`、SimSun、Times New Roman、serif。
- Category / 标签：10px 到 12px，加粗，灰色，全大写或编号化。
- Action Title / 主结论：24px 到 30px，加粗，黑色，行高约 1.3 到 1.4。
- Sub-title / 次级标题：14px 到 16px，加粗，黑色。
- Body / 注释：12px 到 14px，深灰色，常用 bullet。
- Source / 数据来源：10px，浅灰色，位于页面底部或侧栏信息区。

## UI 元素规范

- 边框：常规分割使用 `1px #E5E7EB`，重要表头可使用深色线。
- 容器：直角或 `2px` 小圆角，无阴影，仅用边框或浅背景区分。
- 图表：Chart.js，主色使用深藏青，辅助色使用亮天蓝与灰色。
- 导航：侧边目录只负责滚动定位和当前区块高亮，不作为编辑器外壳。

## 入库规范

- 可编辑文字统一使用 `data-editable="true"`、`contenteditable="true"` 和 `data-slot`。
- 图表容器统一使用 `id="chart-[name]"`。
- 图表数据统一集中在底部 `const chartData`。
- 原始交付文件保存在 `originals/`。
