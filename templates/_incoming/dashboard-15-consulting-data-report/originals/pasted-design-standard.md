# 咨询级数据报告设计标准

## 核心设计原则

- 结论先行：每个模块 / 页面的大标题必须是一句完整的、带有结论的话。
- 极度克制：严禁使用大面积彩色背景、弥散阴影、大圆角、毛玻璃。
- 信息分栏：采用严格的栅格化排版，通常是左侧图表 / 数据 + 右侧核心洞察。
- 引导线：大量使用细实线和细虚线来引导视线。

## 色彩系统

- 主背景：`#FFFFFF`
- 弱强调背景 / 斑马线：`#F8F9FA` 或 `#F0F4F8`
- 主标题 / 核心结论：`#000000` 或 `#111827`
- 正文 / 注释：`#4B5563` 或 `#6B7280`
- 主强调色：`#0F2C59`
- 次强调色：`#00B4D8` 或 `#2563EB`
- 对比 / 辅助色：`#374151`、`#D1D5DB`
- 警示色：`#991B1B`

## 字体与排版层级

- 默认中文字体优先使用无衬线体，如黑体、微软雅黑、PingFang。
- 核心结论可采用粗衬线体，如宋体，以增加权威感。
- Category / 标签：10px 到 12px，全大写或加粗，无衬线，深灰色。
- Action Title / 主干结论：24px 到 28px，粗衬线体或极粗无衬线体，黑色，行高 1.3。
- Sub-title / 次级标题：14px 到 16px，加粗无衬线，黑色。
- Body / 图表注释：12px 到 14px，常规无衬线，深灰色，通常以 bullet points 呈现。
- Source / 数据来源：10px，常规，浅灰色，位于页面最底部。

## UI 元素规范

- 边框：`1px #E5E7EB` 或表头分割使用 `#000000`。
- 卡片 / 容器：直角 `border-radius: 0` 或极微小圆角 `2px`。
- 按钮 / Tag：扁平化，深色背景纯白字，或线框样式。
