# PowerBI HR Operations Report 设计规范

本模板定位为低饱和蓝灰背景、深海军蓝控制面板、白色高密度卡片的人力资源经营分析报告。整体风格偏 Power BI 商务精简版，强调数据密度、硬朗边界和轻量交互。

## 视觉标准

1. 全局背景使用 `#EBF1F6`。
2. 顶部控制面板使用 `#2C3E50`，底部页签栏使用更深的 `#1E2D3B`。
3. 数据卡片使用 `#FFFFFF`，边框使用 `#DDE3EA`，阴影控制为 `0 2px 8px rgba(0,0,0,0.04)`。
4. 普通卡片圆角以 4px 为主，页签和顶栏允许使用小圆角。
5. 主色为 `#2C3E50`，重点留存/趋势高亮使用 `#D97706` 或 `#EAB308`，风险/女性指标使用 `#C0392B`。

## 排版标准

1. 字体优先使用 Inter、PingFang SC、Microsoft YaHei、系统无衬线字体。
2. 标题保持紧凑，图表标题使用 12px 左右字号。
3. 模块间距以 12px 到 16px 为主，卡片内边距以 12px 到 16px 为主。
4. 数值使用 K/M 大写单位，百分比保留两位小数。

## 图表标准

1. 图表统一使用 ECharts。
2. 柱状图、折线图、散点图、漏斗图、雷达图、环图、瀑布图、热力图均使用商务冷暖色阶。
3. Workforce 分布支持 Headcount 与 Proportion 切换；Proportion 视图用 10x10 Heatmap 模拟 Waffle 网格。
4. 页签和图表/表格切换后必须调用 `resize()`，避免隐藏容器导致图表宽度为 0。

## 禁忌

1. 不使用大面积渐变背景。
2. 不使用重阴影和过大圆角。
3. 不使用剧烈缩放、旋转或闪烁动画。
4. 不出现 Lorem ipsum 占位符。
5. 不让图表和文本撑破卡片边界。

## 入库检查

1. 原始 HTML 与粘贴附件完全一致。
2. 实际可编辑文字槽位为 7 个。
3. 实际 `data-section` 为 8 个。
4. 实际 `chart-*` 图表容器为 12 个。
5. 原始模板已包含页签切换后的 `resize()` 保障。
