# 企业级数据分析报告设计规范

## 1. 设计概述与适用场景

本规范旨在创建一套专业、严谨、具有高度信息传达效率的企业级数据报告模板。视觉风格源自成熟实体行业年度报告，强调数据真实性和权威性。

适合场景：

- 企业年度/季度经营分析报告
- ESG 可持续发展报告
- 管理层战略复盘看板
- 向外部投资者或董事会展示的正式汇报

不适合场景：

- C 端用户个性化数据周报
- 实时监控大屏
- 强调科技感、未来感的 AI 概念展示

## 2. 视觉风格定义

核心关键词：专业、清晰、叙事性、克制。

- 去 AI 化：严禁使用大面积紫/蓝渐变背景、霓虹光晕、复杂磨砂玻璃或赛博朋克元素。
- 实体感：页面应类似高级纸质报告，背景以亮灰/白色为主，模块通过浅阴影或细边框区分。
- 信息图表化：图表应结合图标、大数字卡片、流程图等形式，形成数据叙事。

## 3. 色彩体系

- 主背景：`#F4F7F9`
- 模块背景：`#FFFFFF`
- Primary Teal：`#00A99D`
- Secondary Blue：`#1B365D`
- Auxiliary：`#F2C94C` / `#F2994A`
- 主标题/核心数字：`#333333`
- 正文/图表标签：`#666666`
- 辅助说明文字：`#999999`

## 4. 排版与字体

- 字体：`Helvetica Neue`、`Arial`、`PingFang SC`、`Microsoft YaHei`
- H1：32px - 36px，Bold，`#1B365D`
- H2：24px - 28px，Semi-Bold，`#333333`
- KPI：48px - 60px，Bold
- Body：14px - 16px，Regular，行高 1.6
- 图例/轴标签：12px - 13px

## 5. 图表与交互

- 去除不必要网格线、复杂背景和 3D 效果。
- 图表系列颜色严格遵循色彩体系。
- 所有图表必须具备 Hover 交互。
- Tooltip 应白底、简洁、清晰。

## 6. 中文排版禁忌

- 正文段落不要两端对齐。
- 不使用毛笔字、卡通字体或不合适的衬线体。
- 中西文混排注意空格。
- 图表中文标签避免过长导致重叠。
