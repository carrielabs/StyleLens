# Design Standard

## 模板定位

`dashboard-10-executive-logic-report` 是面向 Excel 数据解析后的高管汇报型数据报告模板。视觉重点是克制、清晰、结论先行，适合经营分析、业务复盘、OKR 追踪和管理层汇报。

## 色彩规范

- 主色：深宇航蓝 `#18319D`，用于报告标题、关键边框、主图表线条和核心 KPI 卡片。
- 强调色：活力橙黄 `#F5A623`，用于重点数据、排名第一、关键拐点和高亮标记。
- 页面背景：浅灰蓝 `#F4F6FA`，避免纯白大底。
- 强调背景：极浅蓝 `#F0F4FF`，用于核心观点摘要。
- 文字颜色：核心标题 `#1F2329`，正文 `#4B535E`，辅助说明 `#8F959E`。
- 语义色：增长 `#14B56F`，下降或风险 `#F53F3F`。

## 字体与字号

- 字体：`PingFang SC`, `Microsoft YaHei`, system sans-serif。
- H1 报告标题：28px 左右，Bold。
- H2 模块标题：18px 左右，Bold。
- 数据强调：36px 左右，Bold 或 DIN 风格数字字体。
- 正文：14px。
- 注释、图表坐标轴、图例：12px。

## 卡片与布局

- 页面容器使用 7xl 宽度，模块间距保持 24px 左右。
- 卡片背景为白色，圆角控制在 6px-8px。
- 阴影保持轻量，hover 仅用于增强层级，不作为主要信息表达。
- 核心内容采用四列 KPI、两列图表、流程和明细表组合，适合横向扫描。

## 模块规律

- 结论先行：顶部必须保留 `key-takeaways`。
- 上数下图：先展示 KPI，再进入趋势、排行、漏斗和拆解。
- 多维分析：用雷达图、堆叠面积图表达客群与结构变化。
- 路径复盘：用箭头流程表达业务执行步骤。
- 明细下钻：底部保留数据表，方便承接 Excel 原始明细。

## 入库修正

Gemini 原始 HTML 已具备 `data-section` 和 `chart-*` 容器，但缺少项目标准需要的 `contenteditable="true"` 和 `data-slot`。入库版已为 71 个 `data-editable="true"` 文本补齐 `contenteditable` 与连续 `data-slot="text.xxx"`，并替换了通用占位文案。
