FUI HTML 模板槽位及注入指南 (Template Slots Guide)

为了让 Codex 在注入真实内容时不破坏 FUI 的排版，请遵循以下模块定义进行数据映射：

1. 核心模块 (Header & Hero)

data-section="header": 顶部导航。适合放产品 Logo、版本号、主要功能锚点。注意事项：时间戳和状态码（如 SYSTEM.ONLINE）请原样保留，不要替换为中文。

data-section="hero": 第一屏。必须极其简短、有冲击力。主标题 (H1) 最好是产品英文名或极简中文词（2-4字），副标题用于解释说明。两个按钮分别映射为“主要转化 (DEMO/注册)”和“次要操作 (文档/GitHub)”。

2. 论述模块 (问题与定义)

data-section="pain-points" (痛点): 适合承载“为什么传统方案不行”或“目标用户面临的三大问题”。由于带有红色警告视觉，文案需具有一定的“危机感”。

data-section="status" (定义侧边栏): 适合放产品的核心一句话介绍 (Pitch)。下方的假状态条 (NEURAL CORE) 用于装饰，不要注入真实的业务数据，保留英文最好。

data-section="features" (特征矩阵): 适合放产品的 3-4 个具体功能。文案建议：标题 4-8 个中文字，描述 20-30 个中文字，过长会导致卡片高度不一。

3. 技术与架构展示

data-section="showcase" (终端映射): 一个纯代码画出来的假 UI 界面。适合引导用户去体验真实产品。无需注入大量文本，重点是改动那个 Hover 出现的按钮文案（如“进入控制台”）。

data-section="functions" (战术引擎): 适合列出 4 个底层技术能力（如：高并发、强加密、低延迟等）。短平快，无需长篇大论。

data-section="business-logic" (数据流): 适合展示“产品是如何工作的”的 4 个步骤。步骤说明必须极其简短（6个字以内）。

data-section="architecture" (架构层级): 适合从底层到应用层的逻辑拆解。建议采用“英文代号 + 中文释义”的混合写法，以保持科技感（如 感知层 (SENSORY)）。

4. 深度论述与证明

data-section="core-selling-points" (核心战术能力): 适合放产品最具竞争优势的 3 个卖点。这里是全站唯一适合放置较长文案的地方（每段可容纳 60-80 字）。

data-section="product-value" (效能指标): 纯数据驱动模块。必须是极具冲击力的巨大数字（如 +300%, -80%, 99.9%）。

data-section="analytics" (分析波形): 强烈建议：这里的数字和指标（如 MB/s，PACKETS SENT）当作纯装饰保留，千万不要注入成“累计注册用户”这种毫无 FUI 关联感的词汇。

5. 转化与收尾

data-section="cta" (命令行终端): 将营销话术包装成命令行。例如：把“欢迎注册体验”改成 > 准备就绪。是否将此引擎部署至您的服务器？，极大地增强代入感。

data-section="social-media" & "footer": 外部链接和页脚。保留大量中英文混合的装饰词，只替换链接本身的名称即可。

隐藏与填充规则

模板采用 Flex / Grid 布局。如果有哪个业务模块（如 business-logic 或 functions）你没有合适的内容可填，请直接删除对应的 <section> 标签。

绝不能为了凑数而强行填入与硬核科技无关的废话，这会直接摧毁 FUI 的高质感。
