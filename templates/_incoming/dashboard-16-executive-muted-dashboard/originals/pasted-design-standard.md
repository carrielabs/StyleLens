# 企业级数据分析 Dashboard 设计规范 (Executive Muted Theme)

## 1. 设计核心理念

- 信息密度优先：摒弃大面积留白和巨大的圆角卡片，采用紧凑的网格化布局，适合同屏展示多维数据。
- 克制的视觉表达：拒绝“AI味”的流光溢彩、渐变色、科技蓝。采用纸质/羊皮纸质感的暖色调底色，模拟实体专业商业报告的质感。
- 数据为主，容器为辅：极弱化卡片边界，仅使用 1px 极细边框或微小的色差区分模块，避免容器抢夺数据焦点。

## 2. 适用场景

- 适合：月度/季度经营复盘会、管理层大盘 Dashboard、财务与运营深度分析报告、B2B 业务多维指标监控。
- 不适合：对外 C 端用户的趣味性数据盘点、实时大屏监控、极简的单指标汇报。

## 3. 色彩规范

- 页面背景：`#efeae1`
- 卡片背景：`#fbf7f1`
- 边框与分割线：`#e2dcd0`
- 主文本：`#3c4043`
- 次文本/图例：`#8b939c`
- 主色 1：`#7a9fb1`
- 主色 2：`#d8a47f`
- 主色 3：`#b07f7f`
- 主色 4：`#8ba892`
- 上涨：`#c26d6d`
- 下跌：`#7ba884`

## 4. 排版与字体

- 主字体：系统无衬线体。
- 指标数值：使用高对比度、略粗字重，并开启等宽数字。
- 标题排版：英文字母全大写，中文小字号加粗。
- 禁忌：避免超过 3 种字号层级；严禁使用花体或圆体。

## 5. 组件与模块规范

- Key Takeaway 必须位于顶部或最显眼位置。
- 所有图表必须配备 Tooltip。
- 折线图需包含数据指示线。
- 模块内部间距控制在 12px 到 24px。
- 模块之间控制在 24px 到 32px。

## 6. 中文排版与禁忌

- 中英文之间、中文与数字之间需保留半角空格视觉间隙。
- 不允许出现“请输入”、“测试数据”、“Lorem ipsum”。
- 数据缺失时显示 `-` 或隐藏维度，不显示 `null`。
