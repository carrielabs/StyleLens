# Blueprint Agent Platform 工程蓝图智能体官网设计标准

## 核心定位

本模板定位为工程蓝图风官网，服务于企业级 AI 基础设施、智能体平台、大模型网关和底层技术产品。

## 颜色系统

1. 主背景：`#0012e8`，高饱和度纯蓝。
2. 主文本：`#FFFFFF`。
3. 次文本：`rgba(255, 255, 255, 0.65)`。
4. 弱文本：`rgba(255, 255, 255, 0.4)`。
5. 边框与网格：`rgba(255, 255, 255, 0.1)` 到 `rgba(255, 255, 255, 0.4)`。

## 字体规则

1. 标题与艺术字：`Noto Serif SC, Songti SC, Times New Roman, serif`。
2. 参数、标签、说明：`JetBrains Mono, Courier New, Courier, monospace`。
3. 正文：系统 sans-serif 字体。
4. Hero 标题使用 5rem 以上的大字号，形成学术排版张力。
5. Mono 文案保持小字号与大写感，制造工程参数气质。

## 图纸元素

1. 全局使用 `border-radius: 0` 的硬朗边界。
2. 关键模块使用 CSS 网格背景，模拟工程蓝图。
3. Hero 与方案模块使用 `+` 作为坐标锚点。
4. 背景装饰字使用极低透明度大字，不参与交互。

## 交互规则

1. 编辑态默认不显示破坏视觉的 hover 框，聚焦时才显示提示。
2. 按钮使用反色切换。
3. 卡片 hover 时轻微上移，边框提亮，背景出现低透明白色。
4. 动效统一使用 `0.3s cubic-bezier(0.4, 0, 0.2, 1)` 附近的干净过渡。

## 页面结构

1. `nav`：品牌和四个导航入口。
2. `hero`：背景装饰字、技术标签、标题、说明、双 CTA。
3. `metrics`：四个大数字背书。
4. `target_users`：三类目标用户。
5. `problem`：三类工程化痛点。
6. `solution`：三项架构解决方案。
7. `highlights`：四项核心亮点。
8. `workflow`：三步接入流程。
9. `business_flow`：四段业务流转拓扑。
10. `features_visuals`：功能说明与控制台视觉。
11. `use_cases`：三类使用场景。
12. `roadmap`：产品演进路线。
13. `social`：社区数据与引言。
14. `pricing`：三档价格方案。
15. `cta` 和 `footer`：最终转化与页脚链接。

## 入库调整

1. 保留用户提供的官网 HTML 作为模板源码。
2. 原始交付文件保存到 `originals/`。
3. 生成页中没有真实依据的强事实模块不自动编造。
4. 不启动浏览器、不截图、不做视觉测试。
