THE DILEMMA

工程化落地的深水区困境

ISSUE_01

算力与模型的割裂

企业内部存在多种异构算力与开源闭源模型，缺乏统一的调用与计费网关，导致研发资源严重浪费。

ISSUE_02

黑盒化的推理过程

业务层无法直接干预大模型推理中间态，Token 消耗不可控，且无法满足金融级可追溯审计要求。

ISSUE_03

数据隔离机制匮乏

将高密核心数据直接输入给外部 API 存在合规风险，企业急需一套能在物理隔离环境下运行的检索引擎。

EASY INTEGRATION

3 行代码完成接入

01.

启动本地/私有网关

拉取 Docker 镜像并在一分钟内启动核心调度层。

$ docker pull nous/core-agent
$ docker run -d -p 8080:8080 nous/core-agent
02.

配置模型端点

在控制台填入你已有的商业 API Key 或本地模型地址。

router:
  models:
    - name: gpt-4
    - name: llama-3-local
03.

修改业务代码 BaseURL

无需重构业务代码，只需将原本请求大模型的 BaseURL 指向本地网关即可。

client = OpenAI(
  base_url="http://localhost:8080/v1"
)
